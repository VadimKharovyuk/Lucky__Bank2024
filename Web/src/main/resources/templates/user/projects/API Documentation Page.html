Project Management API Documentation

This documentation provides information about the endpoints available for managing projects and API keys.

Endpoints

GET /projects/new

Displays the form for creating a new project.

Authentication required: Yes

Response: HTML page with project creation form

POST /projects

Creates a new project.

Authentication required: Yes

Request body: ProjectCreateRequest object

ProjectCreateRequest Structure:


public class ProjectCreateRequest {
    private String title;
    private String description;
    private String tokenType;
    private Long userId;
}

Fields:

  • title: The title of the project
  • description: A description of the project
  • tokenType: The type of token for the project
  • userId: The ID of the user creating the project (automatically set by the server)

Response: Redirects to the project details page

GET /projects/{apiKey}

Retrieves details of a specific project.

Authentication required: Yes

Path parameter: apiKey - The API key of the project

Response: HTML page with project details

GET /user/projects

Retrieves all projects for the authenticated user.

Authentication required: Yes

Response: HTML page with a list of user's projects

Authentication

This API supports both OAuth2 and username/password authentication. The API key generated for a project should be included in the header of subsequent API calls for payment-related operations.

Error Handling

In case of errors (e.g., project not found, user not authenticated), appropriate error messages will be displayed on the corresponding HTML pages.

For more information or support, please contact the API administrator.
